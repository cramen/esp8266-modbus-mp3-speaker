# ESP8266 Modbus RTU MP3 Speaker

Прошивка для ESP8266 (NodeMCU), реализующая Modbus RTU Slave для управления MP3-плеером DFPlayer Mini через шину RS-485.

## Технические характеристики
- **Платформа:** ESP8266 (Arduino core).
- **Протокол управления:** Modbus RTU (Slave).
- **Интерфейс связи:** UART0 (Hardware Serial) + MAX485.
- **Интерфейс плеера:** SoftwareSerial (GPIO12/13).
- **Питание:** 5V (рекомендуется раздельное для плеера и контроллера при высоких мощностях).

## Аппаратное подключение
- **MAX485 DE+RE:** GPIO14 (D5).
- **DFPlayer RX:** GPIO13 (D7) через резистор 1кОм.
- **DFPlayer TX:** GPIO12 (D6).
- **RS-485:** UART0 (TX/RX) используется для Modbus. Wi-Fi логгирование отключено.

## Карта регистров Holding (RW)
| Адрес | Параметр | Описание |
| :--- | :--- | :--- |
| 1 | PLAY_CONTROL | 1 = запуск трека из регистра 4 |
| 2 | LOOP_CONTROL | 1 = зацикливание трека из регистра 4 |
| 3 | STOP_CONTROL | 1 = остановка воспроизведения |
| 4 | TRACK_NUMBER | Номер трека (1-9999) |
| 5 | VOLUME | Громкость (0-30) |
| 6 | EQ_MODE | Эквалайзер (0-5) |
| 7-10 | CONFIG | Настройки Modbus (Baud, Parity, Stop bits, ID) |
| 11 | REBOOT | 1 = перезагрузка устройства |
| 12 | OTA_MODE | 1 = включение Wi-Fi AP для прошивки |

## Управление через Coils (WO)
- **Адреса 1-200:** Запись `0xFF00` (ON) инициирует немедленное воспроизведение трека с соответствующим номером.

## Режимы работы
1. **Normal:** Wi-Fi выключен. Активен Modbus Slave на UART0.
2. **OTA:** Активируется через HR 12. Поднимается точка доступа `cr_dr_bl` (пасс: `password`). Веб-интерфейс доступен по адресу `192.168.4.1` для загрузки .bin файла.

## Зависимости
- `DFPlayerMini_Fast`
- `SoftwareSerial` (ESP8266)
- `EEPROM` (встроенная)
